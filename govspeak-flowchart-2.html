GovSpeak — Process Flowchart

End-to-end data flow · User input → AI pipeline → Rendered output
⬤   START   ⬤
Step 1 · UI Layer
🖥️ Open GovSpeak in Browser
Single-file HTML app loads in browser.
Fonts: Playfair Display · Jakarta Sans · JetBrains Mono (Google Fonts).
No server or build step required.
Step 2 · Document Input
📝 Provide Government Document
User chooses one of three input methods:
 ① Paste text into <textarea>
 ② Upload .txt / .pdf file via FileReader API
 ③ Click a built-in sample (Regulation / Tax / Permit / Welfare)
Step 3a · Config
⚙️ Select Simplification Level
grade5 · Grade 5 reading level
plain · Clear adult plain English
teen · Friendly Grade 8 tone
professional · Executive summary
Step 3b · I18N
🌐 Select Output Language
Choose from 24 languages via
<select> dropdown or quick chips.
Default: none (English only).
Flags mapped per locale.
Step 4 · Trigger
▶ Click "Simplify & Translate"
Button fires runGovSpeak(). Also triggered by Ctrl/Cmd + Enter.
Button enters loading state · progress bar animates (indeterminate).

Step 5 · LLM Call [1/2] · Agent 1
🤖 Simplification — DeepSeek-V3
POST https://genailab.tcs.in/v1/chat/completions
Model: azure_ai/genailab-maas-DeepSeek-V3-0324
System: GovSpeak identity prompt
User: document text + chosen level instructions
Params: max_tokens: 2048 · temperature: 0.25
Step 6 · Parse Response
🔍 Extract Structured Fields
Regex extraction from raw LLM text:
 • SIMPLIFIED: → plain language body
 • KEY_TAKEAWAY: → 1–2 sentence summary
 • SCORE: → Flesch-Kincaid readability (0–100)
